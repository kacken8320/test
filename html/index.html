Overview¶

This document introduces low-level systems programming concepts using C. We focus on memory, processes, and performance.

Note

Systems programming trades safety for control. You are responsible for correctness.

Memory Model¶

C gives direct access to memory via pointers.

Important

Undefined behavior is not a runtime error. The compiler may assume it never happens.

Pointers¶

A pointer stores the address of an object.

int x = 42;
int *p = &x;

printf("%d\n", *p);

Dereferencing p accesses the value at that address.

Alignment¶

Most architectures require aligned memory access.

Warning

Misaligned access may crash or silently degrade performance.

Process Model¶

Processes are isolated execution contexts.

Key idea

A process has: - virtual address space - file descriptors - execution state

Creating Processes¶

Processes are created with fork().

#include <unistd.h>

pid_t pid = fork();

if (pid == 0) {
    // child
} else {
    // parent
}

The return value distinguishes parent from child.

File Descriptors¶

int fd = open("file.txt", O_RDONLY);

Math: Cost of a System Call¶

System calls are expensive compared to function calls.

\[T_{syscall} = T_{trap} + T_{kernel} + T_{return}\]

Where:

  • \(T_{trap}\) is the CPU mode switch

  • \(T_{kernel}\) is kernel execution time

Asymptotically:

\[T_{syscall} \gg T_{function}\]
